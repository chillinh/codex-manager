# Codex Manager v1.0.32

Windows desktop manager for switching Codex ChatGPT accounts, monitoring quota snapshots, running warmups, and controlling Codex Desktop auth state from the app, tray, or local API.

## Files

- `Codex Manager Setup 1.0.32.exe` - Windows installer.
- `LOCAL_API_USAGE.md` - Local API usage guide and PowerShell examples.
- `README.md` - Release notes for GitHub.

## What's New

- Added a local API server for account state, switching, refresh, warmup, repair, Codex Desktop status, and OAuth cancellation.
- Added tray menu account details, quick switch, active account actions, Codex Desktop status, and re-auth controls.
- Keeps the tray alive when the main window is closed.
- Detects whether Codex Desktop is running and reopens it after account switch when needed.
- Prevents File Explorer from opening during switch/reopen flows.
- Improved switch performance by validating and syncing auth state before restarting Codex Desktop.
- Repairs auth drift between Codex Manager store and live `.codex/auth.json` when the live auth is newer.
- Handles expired/reused refresh tokens by starting a repair re-auth flow instead of retrying a dead token.
- Allows cancelling or restarting an unfinished OAuth/repair flow without restarting the app.
- Clears selected Codex Desktop session state before reopening to reduce loading-screen hangs.
- Shows the estimated 10-day re-auth countdown on account cards and account details.
- Fixes Windows taskbar/exe icon by embedding the app icon into the packaged executable.

## Notes

- If a refresh token was already reused, it cannot be silently recovered. Re-authenticate that account once in Codex Manager, then the app will resync the saved account and live Codex Desktop auth.
- If Windows still shows an old taskbar icon after installing, unpin the old shortcut and open/pin the new app from the Start Menu. Windows can cache pinned shortcut icons.
- The local API binds to `127.0.0.1` by default. See `LOCAL_API_USAGE.md`.
