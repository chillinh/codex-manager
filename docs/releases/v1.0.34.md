# Codex Manager v1.0.34

Windows desktop manager for switching Codex ChatGPT accounts, monitoring quota snapshots, running warmups, and controlling Codex Desktop auth state from the app, tray, or local API.

## What's Fixed

- Fixed account switching when Codex app-server reports `requiresOpenaiAuth` even though the saved ChatGPT access token is still valid.
- Switch now uses the saved local auth snapshot as the source of truth when it contains a usable ChatGPT access token and account identity.
- Repair completion can now recover from a missing OAuth callback snapshot by using the current saved account auth snapshot when it is valid for the target account.
- This prevents the repair modal from staying `RUNNING` indefinitely after the user already re-authenticated the account.

## Notes

- Refresh-token reuse is still unrecoverable after both the access token expires and the refresh token is rejected. Re-authenticate the account once in Codex Manager in that case.
- If a stale repair flow is already open, cancel it or restart the repair after installing this version.
- Installer assets are distributed through GitHub Releases.
